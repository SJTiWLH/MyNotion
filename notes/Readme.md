# 📖 吉客云销售单自动处理工具｜使用说明书
> 全自动拆分单据 + 清洗订单号 + 回填原销售渠道｜一键处理✅

## 🎯 工具功能概述
本工具用于 **吉客云导出的销售单查询表格** 自动化处理，无需手动改表，全程一键执行：
1. 自动拆分「国补手工单补发」单据与普通单据
2. 智能清洗网店订单号（自动去除无效后缀）
3. 自动匹配 Sheet1 原始渠道，回填修正 Sheet2 单据渠道
4. 输出最终可直接使用的成品 Excel 文件

---

## 📁 环境与文件要求（必看）
1. 从 **吉客云后台** 导出销售单报表
2. 导出文件必须命名为：`销售单查询.xlsx`
3. 将文件放在 **程序根目录**（和exe同文件夹）

### 📂 目录结构
	程序根目录
	├─ 程序.exe
	└─ 销售单查询.xlsx（需手动放入）

---

## 🚀 运行步骤
1. 吉客云导出报表 → 命名为 `销售单查询.xlsx`
2. 放入程序根目录
3. 双击运行程序
4. 等待控制台自动跑完
5. 获取最终文件 `处理后的文件.xlsx`

---

## 📊 程序文件输出说明
运行后自动生成 **2个文件**

### 1. 中间文件：`全部-销售单查询.xlsx`
- Sheet1：**剔除「国补手工单补发」后的所有普通单据**
- Sheet2：**纯国补手工单补发单据**

### 2. 最终成品文件：`处理后的文件.xlsx`（正式使用）
- Sheet1：原始普通单据（未改动）
- Sheet2：**单号清洗完成 + 原销售渠道已回填** 的最终国补单据

---

## 🔧 核心业务规则（订单号清洗逻辑）
### 规则总结
- 以 `-` 分割单号，**智能去除最后一段无效后缀**
- 严格区分「6位日期开头单号」和「普通长单号」逻辑

### ✅ 处理示例
- `260426-682392349070077` → 原样保留
- `260426-682392349070077-BF` → `260426-682392349070077`
- `6925965609847520557-1` → `6925965609847520557`
- `6925965609847520557` → 原样保留

---

## 🔁 完整执行流程
1. 读取 `销售单查询.xlsx`
2. 按销售渠道拆分为两张 Sheet  
3. 对 Sheet2 国补单据清洗订单号后缀
4. 使用清洗后单号匹配 Sheet1 原始销售渠道
5. 将正确渠道回填至 Sheet2「原单销售渠道」
6. 输出最终成品文件

---

## ⚠️ 注意事项 & 常见问题
1. **文件名绝对不能改**：必须是 `销售单查询.xlsx`
2. 文件不能打开占用，否则读取失败
3. 表格必须存在表头：`销售渠道`、`网店订单号`
4. 控制台样式警告可忽略，**不影响数据结果**

---

本工具完全自动化处理，无需人工干预，
日常工作只需：**导出 → 放目录 → 双击运行**，
直接使用最终文件 `处理后的文件.xlsx` 即可！
