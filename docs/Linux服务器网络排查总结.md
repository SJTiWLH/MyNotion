## 一、本次故障复盘

**故障现象：**Linux 插网线正常，但电脑无法连接服务器、服务器无法上网

**根本原因：**物理网卡已识别（UP），但是**没有获取到IPv4地址**，DHCP未分配IP

**最终解决：**更换网线，手动触发DHCP获取IP，联网恢复正常

---

## 二、Linux 必记网络排查命令（自用高频）

### 1. 查看网卡 & IP 地址

```Plain
ip addr
ip a
```

**解读重点：**

- **inet** = 真实内网IP（可用于局域网连接）
    
- **127.0.0.1** = 本地回环，无效外网/局域网IP
    
- **state UP** = 网线正常、网卡已启用
    
- **state DOWN / NO-CARRIER** = 网线没插好/端口故障
    

### 2. 测试外网连通性

```Plain
ping -c 4 223.5.5.5
```

**作用：**测试服务器能不能上网

**判读标准：**

- **0% packet loss** = 网络正常
    
- 超时/不通 = 无IP、无网关、网络故障
    

### 3. 查看路由表（网关）

```Plain
ip route
```

**核心判断：**

- 有 **default via 192.168.1.1** = 网关正常，可以上网
    
- 无 default 行 = 只能内网，无法外网
    

### 4. 测试局域网网关是否通

```Plain
ping -c 4 192.168.1.1
```

能通：服务器和路由器正常互通

不通：没IP、网线、路由器问题

### 5. 快速查看本机真实内网IP

```Plain
hostname -I
```

直接输出可连接的IP，干净无多余内容

## 三、本次核心修复命令（解决网卡无IP）

**场景：网卡UP、网线插好、但是没有IP**

```Plain
sudo dhclient enp0s31f6
```

作用：强制向路由器DHCP申请IP地址，立刻联网

---

## 四、常用补充排查命令（备用）

#### 1. 单独查看网卡状态

```Plain
ip link show enp0s31f6
```

#### 2. 查看SSH远程连接状态

```Plain
systemctl status ssh
```

#### 3. 查看防火墙状态

```Plain
sudo ufw status
```

#### 4. 临时添加网关（上不了网急救）

```Plain
sudo ip route add default via 192.168.1.1
```

## 五、问题总结&避坑经验

- **网线插上不代表有网络**：只是物理接通，必须拿到IP才能通信
    
- **最常见故障**：网卡UP、无IP → DHCP未分配地址
    
- **排查顺序（标准流程）**：
    
    - ip a 看有没有IP
        
    - ip route 看有没有网关
        
    - ping 网关 测试局域网
        
    - ping 外网 测试上网
        
- **最优长期方案**：路由器绑定MAC固定IP，以后永不变IP、永不失联
